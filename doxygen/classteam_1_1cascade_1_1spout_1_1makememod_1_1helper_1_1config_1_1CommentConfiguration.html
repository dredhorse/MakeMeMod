You shouldn't really change anything in here normally all the things you perhaps want to change is in Configuration.

void team.cascade.spout.makememod.helper.config.CommentConfiguration.initializeConfig ( ) throws ConfigNotAvailableException

Initialize the Config, this should done be AFTER setting the needed values, which can be either values needed for the ENUMs (see the example code) or values needed for the class itself:

Exceptions:
ConfigNotAvailableExceptionwhen the config can not correctly be initialized
See also:
alterConfigCurrent(String)
alterPluginSlug(String)

we do the following:

  • create the config if it doesn't exist writeConfigToFile()
  • load the config to memory loadConfigToMemory(String)
  • fix the most common yaml issues if necessary fixYamlIssues()
  • check if we could read something at all
  • apply the config to the enums configurationReading()
  • set debug and config level correctly
  • check if there is an update available if configured autoCheck()
  • check if the configuration requires an update and saves it if configured reloadConfig()

load the configuration into memory, if there are issues try to fix them. the hopefully fixed configuration is auto loaded again, if it fails again we throw a ConfigNotAvailableException

String team.cascade.spout.makememod.helper.config.CommentConfiguration.versionURL
private

Link to the location of the recent version number, the file should be a text with <version> x.y.rb-Bb for example your properties.yml For a better explanation of the version number scheme please look here: { https://github.com/dredhorse/MakeMeMod/wiki/Versioning} For those people who are interested what the version number means, e.g 3.5.1597-20 x.y.rb-Bb

x = Major Version = Major new stuff or internal rework y = Minor Version = Minor new stuff and bug fixes rb = Recommended Build of Spout b = Development Build Version = if you have development builds for example B7
