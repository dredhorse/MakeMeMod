Detailed Description

ENUM for permissions.
The ENUM supports a has method which returns true if the player has the permission or throws a

Exceptions:
NoPermissionException.
It will also directly parse any permission sets you create, for example you can have the following permissions:
  • hlp.cls.test = which is the base permission or the HelperClass
  • hlp.cls.user = which is the user permission, but also has the hlp.cls.test permission
  • hlp.cls.admin = which is the admin permission, but also has the hlp.cls.test and hlp.cls.user permission

    You can name the permission nodes like you want and put as many permission nodes you like into a permission.
    The permissions will be automatically registered with the server and also dumped into a file called permissions.yml for easy reference.

boolean team.cascade.spout.makememod.permissions.PERMISSIONS.has ( Player  player)

Will return true if the player either has the permission directly or if he has one of the permissions Sets. Please note that those permissions Sets are NOT permission.* but a permission node you define and that those permissions Sets only work with this method or the handle method.
NOTE: You can cause an endless loop if you have one permission node in permissionSet which contains that permission node in a permissionSet again!
